# llm-report-html

**This repo is a skill.** The folder `.claude/skills/llm-report-html/` is the primary deliverable. Everything else (Go CLI, Vite renderer, JSON Schema, recipes) is build infrastructure that produces it.

The skill teaches any compatible Agent — Claude Code, Claude Desktop, Cursor, OpenAI Codex, Gemini CLI, etc. — to render typed JSON to a self-contained, optionally interactive HTML report.

```
report.json (Agent writes)  →  llm-report-html render  →  report.html
                                                          (single file, ~3 MB)
```

Reports are deliverables: open in a browser, attach to email, paste in chat. Optional reactive cells make them interactive without writing JS.

---

## Install

```bash
git clone https://github.com/yansir/llm-report-html.git && cd llm-report-html
make             # build binary + populate skill folder
make install     # copy bin/llm-report-html to ~/.local/bin/
```

**Make the skill globally discoverable** (recommended):
```bash
ln -s "$(pwd)/.claude/skills/llm-report-html" ~/.claude/skills/llm-report-html
```

That single symlink makes the skill available in any project, any Claude / Cursor / Codex session.

Requires Go ≥ 1.22 and Node ≥ 18.

> Windows: run from WSL or Git Bash. The helper scripts (`install-binary.sh`, `validate-mermaid.sh`) are POSIX shell.

> Build order: always start with `make` on a fresh clone. The embedded HTML bundle (`internal/render/html/template.html`) is generated by Vite from `template/` and is intentionally gitignored — `go build ./...` alone fails because the embed source doesn't exist yet.

## Quick start (humans)

```bash
llm-report-html recipe list                       # browse 2 technique case studies
llm-report-html recipe show calculator > my.json  # inspect or adapt
llm-report-html validate my.json
llm-report-html render   my.json -o my.html       # opens by default
```

Modify an existing report:
```bash
llm-report-html extract  out.html -o doc.json
# edit doc.json
llm-report-html render   doc.json -o out.html     # add --no-open for headless runs
```

Full CLI: `llm-report-html -h`. Agent guide: `.claude/skills/llm-report-html/SKILL.md`.

## Single source of truth

The manifest (`internal/schema/manifest.json`) is the canonical contract source. It generates `internal/schema/schema.json`; validators, renderer dispatch metadata, type bindings, and the entire skill folder derive from the generated schema.

```bash
llm-report-html schema --json | npx -y json-schema-to-typescript > types.d.ts
```

Strict validation rejects unknown fields, wrong types, and bad enum values with single-line errors:
```
schema validation failed:
  /sections/0/kind: value must be one of "info","warn","danger","success"
  /sections/3/level: must be <= 4 but found 7
```

`make` regenerates schema, renderer catalog, SKILL.md, and `references/*.md`. Generated skill files are gitignored — every clone reproduces them deterministically.

## Architecture in one paragraph

The renderer dispatches each section by `type` to either a content surface (leaf, e.g., `paragraph`, `table`, `mermaid`) or a layout container (e.g., `tabs`, `details`). Reactivity is handled at the section dispatcher: `collectAllRefs` scans the section, `resolveAllBindings` substitutes cells with current values, the encoding renders against pure literal data. Cell changes trigger re-render of just the affected section. Encodings are pure functions of resolved data — they don't know about cells, bindings, or templates. The skill folder generated by `make skill` is the project's interface to Agents; the binary is the engine they shell out to.

## Design principles

- **Data is the only input surface.** Agents and humans write JSON. They never write HTML, CSS, or JS.
- **HTML is the only deliverable.** Single-file, no network, no toolchain at consumption time.
- **Manifest is the contract source.** All derived artifacts (schema, validators, dispatch metadata, types, skill, docs) generate from `manifest.json`.
- **Bounded expressiveness.** 20 surfaces, 8 JSONLogic operators, no escape hatch to arbitrary code.
- **Single renderer.** HTML is the only deliverable. Unsupported targets fail.
